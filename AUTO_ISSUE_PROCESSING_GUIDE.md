# 🤖 Issue 自动处理指南

## 🎯 已配置的自动处理功能

### 1. **Auto Labeler & Triage** (`auto-labeler.yml`)
- **触发:** Issue创建/编辑时 + 每30分钟定时扫描
- **功能:** 自动分类、优先级标记、欢迎评论
- **特点:** 基于规则的智能标签系统

### 2. **AI Issue Processor** (`issue-processor.yml`)
- **触发:** Issue事件 + 每小时定时 + 手动触发
- **功能:** AI分析、自动回复、可修复性检测
- **特点:** 更智能的内容分析

### 3. **Issue 扫描器** (之前配置的)
- **功能:** 报告生成、趋势分析、警报创建
- **特点:** 数据驱动的洞察和建议

## 🔧 自动处理流程

### 当 Issue 被创建时：
```
1. 用户创建 Issue
2. Auto Labeler 自动触发
   ↓
3. 分析标题和内容
   ↓
4. 自动添加标签：
   - 类型标签 (bug/feature/doc/question)
   - 优先级标签 (critical/high/medium/low)
   - 技术栈标签 (python/web/api等)
   ↓
5. 添加欢迎评论
   ↓
6. 如果是高优先级，标记需要关注
```

### 定时扫描时：
```
1. 每30分钟扫描一次
   ↓
2. 检查所有open issues
   ↓
3. 处理未分类的issues
   ↓
4. 生成分类报告
   ↓
5. 创建报告Issue
```

## 🏷️ 自动标签规则

### 类型检测：
- **🐛 Bug:** `bug`, `错误`, `崩溃`, `crash`, `修复`
- **✨ Feature:** `feature`, `功能`, `新功能`, `enhancement`
- **📚 Documentation:** `文档`, `documentation`, `readme`, `typo`
- **❓ Question:** `问题`, `疑问`, `question`, `how to`, `帮助`
- **🔒 Security:** `安全`, `security`, `漏洞`, `攻击`

### 优先级检测：
- **🔴 Critical:** `紧急`, `urgent`, `critical`, `生产环境`
- **🟠 High:** `重要`, `high`, `尽快`, `重要功能`
- **🟡 Medium:** (默认)
- **🟢 Low:** `低优先级`, `low`, `优化`, `有空再做`

### 技术栈检测：
- **🐍 Python:** `python`, `flask`, `pandas`, `yfinance`
- **🌐 Web:** `web`, `前端`, `界面`, `ui`, `浏览器`
- **🔗 API:** `api`, `接口`, `endpoint`, `rest`
- **🚀 Deployment:** `部署`, `deploy`, `docker`, `ci/cd`
- **📊 Monitoring:** `监控`, `monitor`, `日志`, `grafana`

## ⚙️ 配置说明

### 基本配置（已完成）：
```yaml
# 触发条件
on:
  issues: [opened, edited]  # Issue事件
  schedule: ['*/30 * * * *'] # 每30分钟
```

### 规则配置（在代码中）：
```javascript
// 标签规则示例
const rules = [
  {
    name: 'bug',
    keywords: ['bug', '错误', '崩溃'],
    labels: ['bug'],
    priority: 'high'
  }
];
```

### 权限需求：
```yaml
permissions:
  issues: write  # 需要写权限来修改Issue
  contents: read # 需要读权限来分析代码
```
